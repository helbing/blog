操作系统历史

11 October 2019

第一代（1945-1955）：真空管和穿孔卡片

在计算机的早期，是没有操作系统的，人们是通过穿孔卡片的方式来操作计算机，而且计算机主要使用的是真空管，非常容易烧毁

第二代（1955-1965）：晶体管和批处理系统

随着晶体管的发展，计算机已经比较可靠了，不用在担心烧毁的问题，计算机已经可以长时间运行。但计算机依旧非常昂贵，只有少数大公司，政府部门或者大学才能出得起几百万美金购买，而且这时候的计算机非常大，一般锁在有空调的房间里，称作大型机

然而此时操作计算机依旧很麻烦，因为每次只能执行一个程序，非常浪费时间。这也就催生了批处理系统的出现。当然也有磁带的功劳，因为能储存更多的数据和更加轻量，相比于穿孔卡片

讲回批处理系统，程序员会把穿孔卡片送到输入室，在收集了大约一个小时的 job（理解为一个程序） 之后，这些穿孔卡片会通过 IBM 1401 大型机写入到磁带中。随后，操作员会在 IBM 7094 大型机（相比于 1401 来说性能好计算能力强）中装入一个特殊程序（现代操作系统前身，比如 FMS，FORTRAN Monitor System），操作系统会从送过来的磁带中读入 job 并执行，所有 job 的执行结果会写入到另外一盘磁带上，然后在通过 1401 将结果脱机（不与计算机联机）打印出来（这不是很麻烦吗？主要是 7094 性能比较好，而且非常贵，所以就把它主要用于计算）

第三代（1965-1980）：集成电路和多道程序设计

这个时期由于集成电路（IC，Integrate Circuit）的发展，出现了首台使用了（小规模）集成电路的计算机，System/360。当时的操作系统非常多样，每家厂商不得不针对自己旗下的计算机而研发不同的操作系统。而 IBM 为了解决这个问题研发了 System/360 系列的计算机，尽管这些计算机在性能上有差距，但是它们有了统一的操作系统，OS/360，当然由于为了兼容也导致了 OS/360 的代码极其庞大而复杂（在研发 OS/360 过程中所遇到的问题在 Fred Brooks 写的软件工程经典之作《人月神话》中详细地叙述了）

System/360 相比于上一代计算机，引入了两项非常关键的技术，中断和通道

1）通道就是一种控制一台或多台外部设备的硬件机构，它一旦启动就独立与 CPU 运行，因而做到了输入输出与 CPU 并行工作（不是真正意义上的，因为 CPU 要想知道输入输出有没有完成，必须通过轮训请求）。操作员也就不再需要通过 1401 来进行输入输出和搬运磁带了

2）中断技术就是在硬件设备输入输出完成时，或者硬件发生某种故障时等等情况，相应的硬件会向 CPU 发出信号，这时 CPU 立即停下手头的工作优先处理中断请求，中断请求处理完成后再继续原来的作业

通道和中断这两个技术的结合，使得 CPU 和输入输出设备真正实现了并行工作。此时，多道程序设计的概念才变为了现实。那么什么是多道程序设计呢？多道程序设计是指在内存同时放若干道程序，使它们在系统中并发执行，共享系统中的各种资源。当一道程序暂停执行时，CPU立即转去执行另一道程序

然而多道程序系统本质上还是批处理系统，和批处理系统一样，依旧需要漫长的等待，才能拿到程序结果。程序员希望能更快的得到响应，这也催生了分时系统（timesharing）的出现。它实际上是多道程序系统的一个变体，每个用户都有一个联机终端（那个时候，终端机器通过数据总线与中央机器连接，支持中断和数据并行传输）。分时系统就是将 CPU 的运行分为多个时间片，每个作业只执行一个时间片，然后重新进入调度队列，等待下一次调用

  • 第一代分时系统：单道分时系统。作业主要存放在外存中，内存中仅驻留一个作业，该作业运行一个时间片后，便被调出（即调至外存）再从外存调入下一个作业，同样运行一个时间片。依此类推。典型的代表是CTSS（Compatible Time Sharing System），由 MIT（麻省理工大学）开发
  • 第二代分时系统：多道分时系统。奠定了现代分时操作系统的基础。像是 Linux，Unix 系统等。典型的代表是 MULTICS（Multiplexed Information and Computing Service），由 MIT，通用电气和贝尔实验室联合开发，着眼于建造满足波士顿地区所有用户计算需求的机器。虽然最终在商业上失败了，但是它却带来了两个东西，一是贝尔实验室的 Ken Thompson 开发简化的单用户版 MULTICS，而他的工作促使了 Unix 操作系统的诞生。二是计算服务的概念在20世纪以云计算（cloud computing）的形式回归

既然有分时操作系统，那么也有实时操作系统（RTOS）。实时操作系统广泛应用于金融，军工，航天等领域。如果你想了解下实时操作系统的话，可以玩玩单片机和 FreeRTOS

第四代（1980 年至今）：个人计算机

随着 LSI（大规模集成电路）的发展，在每平方厘米的硅片芯片上可以集成数千个晶体管，个人计算机时代到来了

1974 年，当 Intel 8080 第一代通用8位 CPU 出现时，Intel 希望有一个用于8080的操作系统，Gary Kildall 为它编写了一个基于磁盘的操作系统，CP/M（Control Program for Microcomputer）

1977 年，Digital Research 重写了 CPM，使其可以在使用 8080，Zilog Z80 以及其他 CPU 芯片的多种微型计算机上运行，从而完全控制了微型计算机世界达 5 年之久

在 20 世纪 80 年代，Bill Gates 取得了 IBM 公司的合约，收购了 DOS 操作系统，修改后推出了 MS-DOS 操作系统。同一时期 Steve Job 和他的朋友在看到 GUI 后（Doug Engelbart 发明了图形用户界面，包括窗口，图标，菜单以及鼠标等），受启发开发了带有图形界面的苹果计算机 Lisa，然而太贵导致商业上失败。第二次尝试推出了苹果 Macintosh，不仅比 Lisa 便宜的多，而且用户更友好

受到了 Macintosh 的启发，微软在构建 MS-DOS 的后继产品时，开发了名为 Windows 的 GUI 操作系统，后续推出了 Windows 95，Windows 98，Windows NT，Windows 2000，Windows XP，Windows Vista，Windows 7，Windows 8，Windows 10 等一些我们熟悉和不熟悉的操作系统

许多 UNIX 用户，特别是富有经验的程序员更偏好基于命令的界面而不是 GUI，但是几乎所有的 UNIX 系统都支持由 MIT 开发的称为 X Window 的视窗系统（如众所周知的 X11）。这个系统具有基本的视窗管理功能，允许用户通过鼠标创建，删除，移动和变比视窗。对于那些希望有图形系统的 UNIX 用户，通常在 X11 之上还提供一个完整的 GUI，如 Gnome 或 KDE

第五代（1990 年至今）：移动计算机

第一台真正的手持电话出现在 20 世纪 70 年代，大约 2 斤重，绝对属于轻量级。它被人们爱称为“砖头”（想想大哥大）。虽然在电话设备上将通话和计算合二为一的想法在 20 世纪 70 年代就已经出现了，但第一台真正的智能手机直到 20 世纪 90 年代中期才出现。这部手机就是诺基亚发布的 N9000

随着智能手机变得十分普及，各种操作系统之间的竟争也变得更加激烈，并且形势比个人电脑领域更加模糊不清。现如今谷歌公司的 Android 是最主流的操作系统，而苹果公司的 iOS 也牢牢占据次席，但这并不会是常态，想想曾今的霸主 Symbian 操作系统
