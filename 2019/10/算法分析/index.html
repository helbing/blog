算法分析

27 October 2019

算法分析是对一个算法需要多少计算时间和存储空间作定量的分析。也就是衡量算法的时间复杂度和空间复杂度两个角度进行分析

我们以一个简单 for 为例

int n = 100;
for (int i = 0; i < n; i++) {
	printf("%d\n", i);
}

上面这段代码实际执行完需要多久？实际上我们是不清楚的，除非跑一边代码才知道。但是用实际跑代码来衡量显然不符合实际，因为无法排出其他的影响。一般会采用大 O 表示法来进行衡量，假设执行一行代码的时间为 t。那么上面的代码所需的时间为 100t，但是以后 n 的值可能就不是100了，那么它的增长的数量级就为 n，所以我们认为这段代码的时间复杂度为 O(n)

复杂度由小到大依次为（越小越好）

空间复杂度的衡量使用大 S 表示法，衡量方式和时间复杂度差不多，不过空间复杂度衡量的是存储空间的占用情况。以上面的代码为例，它的空间复杂度为 S(n)

对于一个算法，其时间复杂度和空间复杂度往往是相互影响的。当追求一个较好的时间复杂度时，可能会使空间复杂度的性能变差，即可能导致占用较多的存储空间。反之，当追求一个较好的空间复杂度时，可能会使时间复杂度的性能变差，即可能导致占用较长的运行时间。另外，算法的所有性能之间都存在着或多或少的相互影响。因此，当设计一个算法（特别是大型算法）时，要综合考虑算法的各项性能，算法的使用频率，算法处理的数据量的大小，算法描述语言的特性，算法运行的机器系统环境等各方面因素，才能够设计出比较好的算法。算法的时间复杂度和空间复杂度合称为算法的复杂度
