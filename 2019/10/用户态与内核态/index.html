用户态与内核态

14 October 2019

我们知道不管是什么样的操作最终都会变为 CPU 指令，为了保证操作系统的稳健运行，人为的将一组指令定为特权指令，相对应的也划分出了内核态（Kernel Mode）和用户态（User Mode），只有内核态才有权限能操作这组特权指令。在实际中 CPU 会将指令划分到不同的特权等级中，比如 intel 的 x86 处理器有4个特权等级，RING0-3，特权等级依次递减

在 Linux 操作系统中用户态工作在 RING3 中，内核态工作在 RING0 中，高等级的层有权限操作低等级的层，而低等级的层试图操作高等级的层则会显示“非法指令”的错误信息

特权指令涉及如下几个方面

  1. 有关对I/O设备使用的指令，如启动 I/O 设备指令，测试 I/O 设备工作状态和控制I/O设备动作的指令等
  2. 有关访问程序状态的指令，如对程序状态字（PSW）的指令等
  3. 存取特殊寄存器指令，如存取中断寄存器，时钟寄存器等指令
  4. 其他指令

内核态和用户态的切换

1）系统调用

2）异常事件

当 CPU 正在执行运行在用户态的程序时，突然发生某些预先不可知的异常事件，这个时候就会触发从当前用户态执行的进程转向内核态执行相关的异常事件，典型的如缺页异常

3）外围设备中断

当外围设备完成用户的请求操作后，会向 CPU 发出中断信号，此时，CPU 就会暂停执行下一条即将要执行的指令，转而去执行中断信号对应的处理程序，如果先前执行的指令是在用户态下，则自然就发生从用户态到内核态的转换
