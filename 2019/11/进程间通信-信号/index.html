进程间通信-信号

14 November 2019

信号（signal），又称为软中断信号，用来通知进程发生了异步事件。信号的特点是简单，不能携带大量信息，满足特定条件发生。一个信号有以下部分组成

  1. 编号
  2. 名称
  3. 事件，即引起信号产生的事件
  4. 默认处理动作，即信号产生后的行为

我们可以通过 kill -l 命令获取到操作系统提供的信号的编号和名称

信号默认的处理动作有5个

  1. Term，终止进程
  2. Ign，忽略信号
  3. Core，终止进程并产生 core
  4. Stop，暂停进程
  5. Cont，重新运行进程，如果进程被停止

下面的信号是符合 POSIX.1-1990 标准的

Signal Value Action Comment
SIGHUP 1 Term Hangup detected on controlling terminal or death of controlling process
SIGINT 2 Term Interrupt from keyboard
SIGQUIT 3 Core Quit from keyboard
SIGILL 4 Core Illegal Instruction
SIGABRT 6 Core Abort signal from abort(3)
SIGFPE 8 Core Floating-point exception
SIGKILL 9 Term Kill signal
SIGSEGV 11 Core Invalid memory reference
SIGPIPE 13 Term Broken pipe: write to pipe with no readers; see pipe(7)
SIGALRM 14 Term Timer signal from alarm(2)
SIGTERM 15 Term Termination signal
SIGUSR1 30,10,16 Term User-defined signal 1
SIGCHLD 20,17,18 Ign User-defined signal 2
SIGCONT 19,18,25 Cont Continue if stopped
SIGSTOP 17,19,23 Stop Stop process
SIGTSTP 18,20,24 Stop Stop typed at terminal
SIGTTIN 21,21,26 Stop Terminal input for background process
SIGTTOU 22,22,27 Stop Terminal output for background process

需要注意的是，信号 SIGKILL 和 SIGSTOP 这两个信号不能被捕获，阻塞或忽略。其他的信号这里不做介绍，可以通过 man 7 signal 这个命令进行查看

在进程的 PCB 中存在着未决（pending）信号集和阻塞（block）信号集，信号集本质上就是个64位大小的位图，从1-64代表着每一个信号

  • 当某个信号被进程设置为阻塞时，在阻塞信号集中对应的位就会被设置为1，代表对应信号被阻塞
  • 当某个信号被进程捕获后，在未决信号集中对应的位就会被设置为1，代表对应信号被未决，等待进程处理，进程在处理之前会检查阻塞信号表，如果对应信号没有被阻塞，则处理信号，处理结束后，未决信号集中对应的位设置为0

我们知道在终端中使用 ctrl+c 能产生 SIGINT 信号终止进程。现在我们来写一个例子，将 SIGINT 设置为阻塞状态。使用到的函数是 sigprocmask，它的函数签名如下

int sigprocmask(int how, const sigset_t *set, sigset_t * oldset);
  • how
    • SIG_BLOCK，传入的信号集与现有的阻塞信号集做或操作，结果作为新的阻塞信号集
    • SIG_UNBLOCK，传入的信号集与现有的阻塞信号集做与操作，结果作为新的阻塞信号集
    • SIG_SETMASK，传入的信号集直接替换现有的信号集
  • set，传入的信号集
  • oldset，旧的信号集
  • 返回值，成功返回0，失败返回-1，失败原因在 errno 中

下面是一个简单的例子

int main()
{
    // 创建一个信号集
    sigset_t set;
    // 创建一个信号集，专门用于接收旧的信号集
    sigset_t oldSet;
    // 信号集清零
    sigemptyset(&set);
    // 将对应信号的位设置为1
    sigaddset(&set, SIGINT);
    // 将信号集与现有的阻塞信号集做或操作，结果作为新的阻塞信号集
    // 如果不想知道操作前的阻塞信号集，oldset 传 NULL 就行了
    sigprocmask(SIG_BLOCK, &set, &oldSet);

    for (int i = 0; i < 60; i++) {
        printf("%d\n", i);
        sleep(1);
    }
}

接下来，启动进程后，你在试下看看，键入 ctrl+c 是否还能终止进程

我们在来讲一个我们平常很经常用的命令，kill 命令，我们常用它来杀死进程，其实这个命令是作用是将信号发送给进程，或者用于显示信号列表 kill -l。kill 同样也有对应函数，函数签名如下

int kill(pid_t pid, int sig);
  • pid
    • pid>0，将信号传给进程 id 为 pid 的进程
    • pid=0，将信号传给和当前进程相同进程组的所有进程
    • pid=-1，将信号广播传送给系统内所有的进程
  • sig，信号
  • 返回值，成功为0，失败为-1，失败原因在 errno

下面是一个简单的例子

#include <sys/types.h>
#include <signal.h>
#include <stdio.h>
#include <unistd.h>

int main()
{
    pid_t pid = fork();

    if (pid > 0) {
        // 3秒后，父进程发送一个 SIGKILL 信号给子进程
        sleep(3);
        kill(pid, SIGKILL);
    } else if (pid == 0) {
        while (1)
        {
            printf("I am children\n");
            sleep(1);
        }
    }

    return 0;
}

你可以通过 alarm 给当前进程设置一个信号时钟，设定多少秒后给当前进程发送一个 SIGALRM 信号，alarm 的函数签名如下

unsigned int alarm(unsigned int seconds);
  • seconds，秒数，如果 seconds 为0，则相当于取消闹钟
  • 返回值为之前闹钟的剩余秒数，如果之前未设置闹钟则返回0

为了方便演示例子，我们在讲多一个函数 signal，它的作用是用于设置信号处理行为，替换信号的默认行为。它的函数签名如下

void (*signal(int signum, void(* handler)(int)))(int);

下面是一个简单的例子

#include <sys/types.h>
#include <signal.h>
#include <stdio.h>
#include <unistd.h>

void handler()
{
    printf("接收到信号\n");
}

int main()
{
    // 设置当前进程 SIGALRM 信号的处理行为
    signal(SIGALRM, handler);

    // 3秒后收到 SIGALRM 信号
    alarm(3);

    for (int i = 0; i < 5; i++) {
        printf("%d\n", i);
        sleep(1);
    }
}
// 0
// 1
// 2
// 接收到信号
// 3
// 4
