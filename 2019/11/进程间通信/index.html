进程间通信

12 November 2019

进程间通信（IPC，InterProcess Communication），通过内核系统的缓冲区进行数据交换的机制，大体有以下几种方式

  1. 管道，通常指无名管道，是 UNIX 系统 IPC 最古老的方式，缺点是只能是有血缘关系的进程之间才能使用
  2. FIFO，也称为有名管道，也是 UNIX/Linux 系统的一种文件格式
  3. mmap 文件映射共享 I/O
  4. socket
  5. 共享内存
  6. 消息队列
  7. 信号量

管道

首先需要说明常见的通信方式有三种

  1. 单工（Simplex Duplex），只能读或者只能写，比如广播
  2. 半双工（Half Duplex），同一时间，要么只能读，要么只能写，比如对讲机
  3. 全双工（Full Duplex ），同一时间，既可以读，也可以写，比如电话

管道（无名管道）就是一种半双工通信方式，函数签名如下

int pipe(int fd[2])

通过 pipe 函数申请两个文件描述符，fd[0] 描述符用于读数据，fd[1] 描述符用于写数据。由于 read 函数读数据时会阻塞，这也导致了使用管道通信时，同一时间进程要么处于读数据，要么是写数据，这也是为什么管道是半双工通信的原因。pipe 的返回值，0表示成功，-1表示失败，可以通过 errno 知道错误原因

#include <stdio.h>
#include <unistd.h>

int main()
{
    int fd[2];
    pipe(fd);

    pid_t pid = fork();

    if (pid == 0) {
        printf("fd[1] = %d\n", fd[1]);
        sleep(3);
        write(fd[1], "hello", 5);
    } else if (pid > 0) {
        printf("fd[0] = %d\n", fd[0]);
        char buf[6] = {0};
        // read 函数是阻塞的
        int ret = read(fd[0], buf, sizeof(buf));
        if (ret > 0) {
            write(STDOUT_FILENO, buf, ret);
        }
    }

    return 0;
}
// fd[0] = 3
// fd[1] = 4
// hello%

由于申请新的文件描述符是当前进程中最小可用的文件描述符号码，0，1，2默认被占用，这也是为什么 fd[0] 是3，fd[1] 是4的原因

FIFO

由于管道只能是有血缘关系的进程间才能进行通信，这也催生了 FIFO （有名管道）的出现，FIFO 实现了无血缘进程的通信。FIFO 本质上就是通过 mkfifo 命令或函数创建管道伪文件，内核会针对这个 FIFO 文件开辟一个缓冲区，各进程通过操作这个 FIFO 文件的读写来实现进程间通信

我们先使用终端来体验下有名管道，通过 mkfifo 创建一个管道伪文件

mkfifo myfifo

接下来读取 myfifo 中的内容，由于还没有内容，所以处于阻塞状态

然后我们另启一个终端，然后让 myfifo 中写数据

这时候就能读取到 myfifo 中的数据了

接下来我们看下函数调用 mkfifo，它的函数签名如下

int mkfifo(const char * pathname, mode_t mode)
  • pathname，管道路径名
  • mode，权限

  • 返回值，0表示成功，-1表示失败，失败原因在 errno 中

    #include <stdio.h>
#include <unistd.h>
#include <sys/types.h>
#include <sys/stat.h>
#include <fcntl.h>
#include <string.h>

int main()
{
char * fifoName = "myfifo";
char * message = "Hello World";

mkfifo(fifoName, 0666);

int fd;

pid_t pid = fork();

if (pid == 0) {
    fd = open(fifoName, O_WRONLY);
    write(fd, message, strlen(message));
} else if (pid > 0) {
    fd = open(fifoName, O_RDONLY);
    int len = strlen(message) + 1;
    char buf[len];
    read(fd, buf, len);
    printf("%s\n", buf);
}

close(fd);

return 0;
}
// Hello World
