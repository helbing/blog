进程间通信-mmap

13 November 2019

mmap 用来将某个文件内容映射到内存中，对该内存区域的存取即是直接对该文件内容的读写，mmap 的函数签名如下

void *mmap(void *start, size_t length, int prot, int flags, int fd, off_t offsize);
  • start，指向欲对应的内存起始地址，不过一般填 NULL，让操作系统自动选定地址
  • length，表示申请内存的大小
  • prot，表示映射区域的保护方式
    • PROT_EXEC，映射区域可被执行
    • PROT_READ，映射区域可被读取
    • PROT_WRITE，映射区域可被写入
    • PROT_NONE，映射区域不可读写
  • flags
    • MAP_SHARED，对应射区域的写入数据会复制回文件内，而且允许其他映射该文件的进程共享
    • MAP_PRIVATE，对应射区域的写入操作会产生一个映射文件的复制，即私人的“写入时复制”（copy on write）对此区域作的任何修改都不会写回原来的文件内容
    • MAP_ANONYMOUS，建立匿名映射，此时会忽略参数fd，不涉及文件，而且映射区域无法和其他进程共享
    • MAP_LOCKED，将映射区域锁定住，这表示该区域不会被置换（swap）
    • 在调用 mmap 时必须要指定 MAP_SHARED 或 MAP_PRIVATE
  • fd，映射文件的文件描述符
  • offset，文件映射便宜量，通常设置为0，offset 必须是分页大小的整数倍
  • 返回值
    • 成功返回映射地址的首地址
    • 失败返回 MAP_FAILED，即是 (void *) -1，错误原因在 errno

下面是一个简单的例子

#include <sys/types.h>
#include <sys/stat.h>
#include <fcntl.h>
#include <unistd.h>
#include <sys/mman.h>
#include <stdio.h>

int main()
{
    struct stat sb;
    int fd = open("/etc/passwd", O_RDONLY);

    fstat(fd, &sb);

    void* start = mmap(NULL, sb.st_size, PROT_READ, MAP_PRIVATE, fd, 0);

    if (start == MAP_FAILED) {
        return -1;
    }

    printf("%s", start);

    // 释放 mmap
    munmap(start, sb.st_size);

    close(fd);

    return 0;
}
