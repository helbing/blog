进程间通信-FIFO

12 November 2019

由于管道只能是有血缘关系的进程间才能进行通信，这也催生了 FIFO （有名管道）的出现，FIFO 实现了无血缘进程的通信。FIFO 本质上就是通过 mkfifo 命令或函数创建管道伪文件，内核会针对这个 FIFO 文件开辟一个缓冲区，各进程通过操作这个 FIFO 文件的读写来实现进程间通信

我们先使用终端来体验下有名管道，通过 mkfifo 创建一个管道伪文件

mkfifo myfifo

接下来读取 myfifo 中的内容，由于还没有内容，所以处于阻塞状态

然后我们另启一个终端，然后让 myfifo 中写数据

这时候就能读取到 myfifo 中的数据了

接下来我们看下函数调用 mkfifo，它的函数签名如下

int mkfifo(const char * pathname, mode_t mode)
  • pathname，管道路径名
  • mode，权限
  • 返回值，0表示成功，-1表示失败，失败原因在 errno 中

下面是一个简单的例子

#include <stdio.h>
#include <unistd.h>
#include <sys/types.h>
#include <sys/stat.h>
#include <fcntl.h>
#include <string.h>

int main()
{
    char * fifoName = "myfifo";
    char * message = "Hello World";

    mkfifo(fifoName, 0666);

    int fd;

    pid_t pid = fork();

    if (pid == 0) {
        fd = open(fifoName, O_WRONLY);
        write(fd, message, strlen(message));
    } else if (pid > 0) {
        fd = open(fifoName, O_RDONLY);
        int len = strlen(message) + 1;
        char buf[len];
        read(fd, buf, len);
        printf("%s\n", buf);
    }

    close(fd);

    return 0;
}
// Hello World
